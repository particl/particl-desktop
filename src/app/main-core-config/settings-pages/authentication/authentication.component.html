Core RPC Authentication Applications typically connect to Particl Core using its RPC service. This RPC service usually requires authentication parameters, which can either be generated and obtained automatically (from a cookie file) or from user-provided authentication details.

Use RPC cookie file

When starting a new Particl Core instance, Particl Core can generate single use credentials for connecting to its RPC services. These credentials are read in and made available to Particl Desktop applications without user intervention.

However, if Particl Desktop is not managing the Particl Core node (ie: connecting to a remote Particl Core node instead of starting a new Particl Core node), or custom supplied credentials are wanted to be used, then unselect this option to provide connection details.

If connecting to a remote Particl Core node, but the running Particl Core node is running from either of the default location on the current machine, or from a specified custom data directory, then it may be possible to still read the Particl Core generated cookie file. If authentication fails, it may be necessary to configure the remote node with custom RPC credentials and un-select this option to provide the custom RPC credentials further below.

Use Particl Core cookie file for authentication

RPC Username

Allows for a custom username to be used for RPC connections.

Starting a new Particl Core node starts the RPC service with this username, and Particl Desktop applications using the RPC service will use this username for data requests.

If connecting to an already running Particl Core node, this username will be used to authenticate against the running instances' RPC service.

RPC Username is invalid

RPC Password

Starting a new Particl Core node starts the RPC service with this password, and Particl Desktop applications using the RPC service will use this password for data requests.

If connecting to an already running Particl Core node, this password will be used to authenticate against the running instances' RPC service.
